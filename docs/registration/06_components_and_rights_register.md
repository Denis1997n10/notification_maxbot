# Реестр компонентов, прав и подтверждающих документов

## 1. Назначение

Документ является шаблоном для подготовки сведений о составе программного обеспечения, сторонних компонентах и подтверждении прав. Его нужно заполнить до подачи заявления на регистрацию.

## 2. Сведения о программном продукте

| Поле | Значение |
|---|---|
| Рабочее наименование | Платформа подписок и уведомлений жителей по событиям объектов жилищной инфраструктуры |
| Репозиторий | `Denis1997n10/notification_maxbot` |
| Тип продукта | Программное обеспечение / web + serverless application |
| Основной язык backend | Python |
| Frontend | JavaScript / Vite-based static apps |
| Инфраструктура | Terraform, Yandex Cloud |
| База данных | YDB Serverless |
| Канал уведомлений MVP | MAX |
| Внешний источник MVP | RegionCity/MPOISK |

## 9. Что нельзя делать

- Нельзя заявлять исключительные права без подтверждающих документов.
- Нельзя игнорировать open-source лицензии.
- Нельзя скрывать обязательные внешние зависимости.
- Нельзя включать в документацию функции как реализованные, если они только запланированы.
- Нельзя хранить реальные секреты в этом документе.

## 10. Следующий шаг по этому документу

1. Сгенерировать список Python-зависимостей.
2. Сгенерировать список Node.js-зависимостей.
3. Для каждой зависимости определить лицензию.
4. Уточнить правообладателя собственных компонентов.
5. Привязать каждый собственный компонент к договору/акту/служебному заданию.
6. Подготовить финальную таблицу для юридической проверки.
