# Начальный пакет документации для регистрации ПО

Документы в этой папке подготовлены как стартовый комплект для подготовки продукта к регистрации в Едином реестре российских программ для ЭВМ и баз данных.

Важно: этот комплект не заменяет юридическую проверку правообладателя, состава исключительных прав, структуры владения, лицензионной модели и состава сторонних компонентов. Перед подачей заявления правообладатель должен подтвердить сведения, которые не могут быть выведены из исходного кода: реквизиты организации, правовой режим исходного кода, договоры с авторами/подрядчиками, сведения о бенефициарах, выручке и лицензионных отчислениях.

## Состав стартового комплекта

1. `01_user_manual.md` — инструкция пользователя системы. Это приоритетный документ для демонстрации назначения, ролей и базовых сценариев эксплуатации.
2. `02_functional_characteristics.md` — функциональные характеристики программного обеспечения.
3. `03_technical_description.md` — техническое описание архитектуры, модулей и интеграций.
4. `04_installation_and_operation.md` — инструкция по установке, развертыванию и эксплуатации.
5. `05_registration_readiness_checklist.md` — чек-лист готовности к подаче заявления и список недостающих сведений.
6. `06_components_and_rights_register.md` — заготовка реестра сторонних компонентов, прав и подтверждающих документов.

## Предлагаемая классификация

Текущая система является программным обеспечением, а не программно-аппаратным комплексом: код разворачивается в облачной инфраструктуре и не требует поставки специализированного аппаратного оборудования как составной части продукта.

Предлагаемое рабочее наименование для документов: **Платформа подписок и уведомлений жителей по событиям объектов жилищной инфраструктуры**.

Краткое назначение: программное обеспечение для ведения справочника объектов жилищной инфраструктуры, управления подписками пользователей на объекты, получения событий от внешних учетных систем и отправки уведомлений пользователям через подключаемые каналы коммуникации.

## Нормативная база для дальнейшей проверки

- Постановление Правительства РФ от 16.11.2015 № 1236 — правила формирования и ведения Единого реестра российских программ для ЭВМ и баз данных.
- Официальный сайт реестра: https://reestr.digital.gov.ru/
- Федеральный закон № 149-ФЗ «Об информации, информационных технологиях и о защите информации» — общая правовая база, в рамках которой действует реестр.

## Что еще нужно подготовить до подачи

- Подтверждение правообладателя и исключительных прав на ПО.
- Сведения о правообладателе и структуре владения.
- Сведения о сторонних open-source и коммерческих компонентах.
- Описание жизненного цикла разработки, сборки и поддержки.
- Актуальные скриншоты пользовательского интерфейса и административной панели.
- Демо-доступ или демонстрационный стенд, если потребуется для экспертизы.
- Финальная версия эксплуатационной документации в формате, пригодном для подачи.
