# MAX Bot User Flow

Бот остается универсальным каналом подписок на объекты справочника. В MVP объект подписки - подъезд, но команды не завязаны на уборку как единственный сценарий.

## Основной сценарий

1. Житель открывает QR-код подъезда или публичную страницу адреса.
2. Публичная страница открывает MAX bot через deeplink `${MAX_BOT_DEEPLINK_BASE}?start=e_<public_code>`.
3. Бот сразу подписывает пользователя на подъезд и показывает главное меню.
4. Дальше пользователь управляет подписками через кнопки или простые текстовые команды.

## Выбор адреса в mini app

Если QR-кода под рукой нет, житель нажимает в боте `Выбрать адрес`. Открывается публичное mini app на `/?view=select`:

1. Пользователь выбирает город, район, улицу, дом и подъезд через поля с поиском.
2. Mini app показывает выбранный адрес, QR-код для MAX и две кнопки: `Перейти в MAX` и `Открыть страницу адреса`.
3. Кнопка `Перейти в MAX` открывает deeplink с payload `e_<public_code>`, после чего webhook `bot_started` оформляет подписку в боте.
4. Кнопка `Открыть страницу адреса` ведёт на публичную страницу `/e/<public_code>`.

Для прямой подписки public API использует только подписанные MAX WebApp данные и public code подъезда. Квартиры, фотографии и детальные delivery logs не передаются.

## Команды

- `Мои адреса` - показать активные подписки.
- `Выбрать адрес` - открыть mini app выбора поддерживаемого адреса.
- `Подписаться <код>` - добавить адрес по public code из ссылки.
- `Отписаться <номер>` - отключить адрес по номеру из списка `Мои адреса`.
- `Услуги` - зарезервированный раздел будущих сервисов.
- `Помощь` - краткая памятка по командам.

## UX правила

- Бот не просит квартиру и не хранит квартиры.
- Бот не предлагает ручной поиск чужих адресов; адрес добавляется через QR/deeplink или явный public code.
- Каждое сообщение с выбором действия содержит MAX inline keyboard с callback/link-кнопками; действия с параметрами передаются через callback payload.
- Если команда не распознана, бот возвращает главное меню, а не техническую ошибку.
