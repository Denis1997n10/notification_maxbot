# MAX Bot User Flow

Бот остается универсальным каналом подписок на объекты справочника. В MVP объект подписки - подъезд, но команды не завязаны на уборку как единственный сценарий.

## Основной сценарий

1. Житель открывает QR-код подъезда или публичную страницу адреса.
2. Публичная страница открывает MAX bot через deeplink `/start e_<public_code>`.
3. Бот сразу подписывает пользователя на подъезд и показывает главное меню.
4. Дальше пользователь управляет подписками через кнопки или простые текстовые команды.

## Команды

- `Мои адреса` - показать активные подписки.
- `Подписаться` - объяснить, как добавить адрес через QR или публичную страницу.
- `Подписаться <код>` - добавить адрес по public code из ссылки.
- `Отписаться <номер>` - отключить адрес по номеру из списка `Мои адреса`.
- `Отключить все` - отключить все активные уведомления пользователя.
- `Услуги` - зарезервированный раздел будущих сервисов.
- `Помощь` - краткая памятка по командам.

## UX правила

- Бот не просит квартиру и не хранит квартиры.
- Бот не предлагает ручной поиск чужих адресов; адрес добавляется через QR/deeplink или явный public code.
- Каждое сообщение с выбором действия содержит MAX inline keyboard с кнопками типа `message`, чтобы нажатие превращалось в обычное входящее сообщение.
- Если команда не распознана, бот возвращает главное меню, а не техническую ошибку.
